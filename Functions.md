можно обращаться к атрибутам функции
```Python
>>> def f():
    """123"""
    return 92
>>> f.__name__
'f'
>>> f.__doc__
'123'
```


в питоне можно передавать в функцию позиционные и именованные аргументы.
причем для того чтобы питон смог спарсить значения именованные должны идти после позиционных
 *Пример*
 ```Python
def min(x, y):
	return x if x < y else y

min(1, 2)
min(1, y=2)
min(x=1, y=2)
min(y=2, x=1)
 ```

можно передавать в функцию произвольное количество аргументов
для этого используется \*args 
```Python
def min(*args):
	# type(args) => <class 'tuplr'> 
	res = float('inf')
	for a in args:
		res = a if a < res else res
	return res
```

такой "упаковщик" аргументов можно комбинировать с обычными аргументами

```Python
def min(first, *rest):
	res = first
	for x in rest:
		res = x if x < res else res
	return res
```

вообще упаковщик аргументов в функции упаковывает все оставшиеся позиционные аргменты (то есть после него идут только именованные), а так можно наоборот распоковывать значения

```Python
>>> a = [1, 3, 5]
>>> b = [6, 4]
>>> c = [*a, *b]
>>> c
[1, 3, 5, 6, 4]
```

кроме того в питоне можно передавать дефолтные значения в функциях

```Python
def min(*args, default=None):
	if not args:
		return default
	
	return min(args)
```

но с дефолтными значениями не так все просто. дефолтные значения являются просто атрибутом функции. поэтому если их менять, то изменения удивительным образом сохранятся

```Python
>>> def foo(x=[], y=92):
    x.append(1)

>>> foo.__defaults__
([], 92)
>>> foo()
>>> foo.__defaults__
([1], 92)
```

для решения такой проблемы можно сделать так

```Python
def foo(x=None, y=92):
    x = x or []
    x.append(1)

foo.__defaults__
(None, 92)
foo()
foo.__defaults__
(None, 92)
```

можно сделать так, чтобы все аргументы после передавались только с именем

```Python
def flatten(iterable, *, depth=None):
	pass
```

для произвольного количества именнованных аргументов используется 
`**kwargs`
```Python
>>> def call_me(*args, **kwargs):
	return (args, kwargs)
>>> (args, kwargs) = call_me(1, 2, a=3, b=4)
>>> args
(1, 2)
>>> kwargs
{'a': 3, 'b': 4} # dict
```

`**` - работает как упаковщик и распаковщик словаря.
это всегда нужно передавать в функцию


Если собрать всё, что мы обсудили (позиционные, дефолты, `*args`, именованные и `**kwargs`), получится «мастер-формула» идеальной функции. Порядок **строго фиксирован**:

1. **Позиционные обязательные** (`a, b`)
2. **Позиционные с дефолтами** (`c=3`)
3. **Сборщик позиционных** (`*args`)
4. **Именованные (Keyword-only)** — с дефолтами или без (`d, e=5`)
5. **Сборщик именованных** (`**kwargs`)


## lambda или анонимные функции

нужны для того чтобы не засорять ими основной код и вставлять или передавать в другие функции

```Python
# Обычная функция
def square_def(x):
    return x ** 2

# Lambda-аналог
square_lambda = lambda x: x ** 2

print(square_lambda(5)) # 25
```

после lambda обязательно только одно выражение.

не стоит делать так:
```Python
f = lambda x: expr
```
если мы так делаем то уже лучше просто объявить ее
```Python
def f(x):
	return expr
```

анонимные функции чаще всего нужны в map и filter

map - возвращает объект по которому можно проитерироваться, объекты - примененные функции

filter - ясно по названию

```Python
nums = [1, 2, 3, 4, 5]

# Оставляем только четные (filter)
evens = list(filter(lambda x: x % 2 == 0, nums))

# Возводим всё в квадрат (map)
squares = list(map(lambda x: x**2, nums))
```

zip - делает пары (или даже больше) из нескольких итерируемых объектов

```Python
>>> list(zip("hello", range(10), range(100, 1000)))
[('h', 0, 100), ('e', 1, 101), ('l', 2, 102), ('l', 3, 103), ('o', 4, 104)]
```
