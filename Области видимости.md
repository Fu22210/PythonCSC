замыкание - переменные внешней функции


питон делает оптимизацию, для того чтобы не хранить лишние из замыкания
```Python
>>> def foo():
...     def bar():
...         return x
...     print(bar.__closure__)
...     print(locals())
...     x = 92
...     print(bar.__closure__)
...     print(locals())
... foo()
...
(<cell at 0x101368f40: empty>,)
{'bar': <function foo.<locals>.bar at 0x101359bc0>}
(<cell at 0x101368f40: int object at 0x1019eab60>,)
{'bar': <function foo.<locals>.bar at 0x101359bc0>, 'x': 92}
```
то есть интерпретатор для локальных переменных сначала просматривает переменные которые в теле выделяются, а потом выделяет под них место
есть такой пример

```Python
>>> x = 1
>>> def foo():
...     print(x)
...     x = 2
...
>>> foo()
Traceback (most recent call last):
  File "<python-input-7>", line 1, in <module>
    foo()
    ~~~^^
  File "<python-input-6>", line 2, in foo
    print(x)
          ^
UnboundLocalError: cannot access local variable 'x' where it is not associated with a value
```

под x выделяется уже место в замыкании, поэтому мы не смотрим на глобальную переменную x

правило поиска называется LEGB:

1. **L**ocal (Локальная область).
2. **E**nclosing (Замыкающая / Нелокальная)
3. **G**lobal (Глобальная)
4. **B**uilt-in (Встроенная)

для того чтобы в локальном коде изменять глобальные переменные существуют слова global

но стоит учитывать, что в замыкании мы храним не само значение а место где оно хранится ![[Снимок экрана 2026-03-25 в 11.38.13.png]]
